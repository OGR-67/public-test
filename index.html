Qui suis-je?

À 36 ans, je capitalise déjà 20 années de bons et loyaux services dans l'armée de l'air. Maintenant, le temps du changement est venu.
Mes raisons sont diverses: géographique, mutation, etc...

Ayant toujours travaillé dans le domaine technique, je peux dire sans sourciller que je suis fait pour cela.
Logique, recherche de pannes, esprit d'analyse et autonomie sont parmis les qualités nécessaires à mon quotidien.

S'ajoutent à cela la rigueur, le respect des règles et des procédures ainsi qu'un sens de l'engagement propre au statut de militaire.

Ces attributs, couplés à ma passion pour l'informatique m'ont donc poussé à croire en moi et tenter une reconversion dans ce domaine.

Mon apprentissage en autodidacte, sur mon temps libre, m'a permis d'atteindre un certain niveau de compétence.

Or, ce dernier a ses limites, c'est pour cela que je compte démarrer une formation en alternance pour devenir concepteur / développeur d'applications.

Mes réalisations

Randomecipe

Technologies:
  • Python
  • Django
  • API Call
  • SQLite
  • HTML / CSS / Javascript

Le concept de ce site est de consulter 5 recettes. Chaque semaine, les recettes sont mises à jour, mais une recette qui a été mise dans les favoris reste accessible car ces dernières sont enregistrées dans la base de données. Vous aurez deviné l'existance de comptes utilisateurs.

Ce site fut mon premier projet complet. Il utilise le framework Django. Ce choix fut réfléchi car, par nature, Django nous force à l'utilisation d'une architecture MVT et donc d'avoir une application bien structurée.

Les recettes sont récupérées à l'aide d'une API de recettes aléatoires.

Wedding

Technologies:
  • NodeJS
  • Express
  • EJS
  • MongoDB - Mongoose
  • Bootstrap

Il s'agit d'un site fictif de faire-parts de mariage, proposant une liste d'hôtels générée dynamiquement en interrogeant une API. Il permet de confirmer sa présence, accompagné ou non, sous la forme d'un formulaire d'inscription.

Ce projet avait pour but de tester mon adaptabilité en créant une appli web sous une autre technologie que celles que je connaissais déjà. Au cours de ce projet j'ai donc migré de python à Javascipt avec NodeJS, de Django à Express avec EJS et de SQLite à MongoDB.
